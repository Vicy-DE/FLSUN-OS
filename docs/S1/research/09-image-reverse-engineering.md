## boot.img — Android Boot Image (mkbootimg)

### Header Analysis

The boot.img uses the **Android boot image format** — magic bytes `ANDROID!` (`41 4E 44 52 4F 49 44 21`). This is the standard Rockchip approach for Linux systems on RV1126, using `mkbootimg` to pack the kernel and device tree together.

| Field | Value |
|---|---|
| Magic | `ANDROID!` |
| Header version | 0 |
| Page size | 2,048 bytes |
| Kernel size | 8,531,192 bytes (8.14 MB) |
| Kernel load address | `0x10008000` |
| Ramdisk size | 0 (no ramdisk) |
| Second stage size | 1,538,048 bytes (1.50 MB) |
| Second stage address | `0x10F00000` |
| Tags address | `0x10000100` |
| OS version | Not set |
| Cmdline | Empty (kernel cmdline from U-Boot) |
| Name | Empty |

> **Key correction:** Previous analysis (doc 08) theorized boot.img was a FAT/EXT2 partition with `extlinux.conf`. This is wrong — it is a binary-packed Android boot image. Rockchip uses this format even for Linux systems.

### Component Layout

| Offset | Size | Content |
|---|---|---|
| `0x000` – `0x7FF` | 2,048 B | Android boot image header |
| `0x800` – `0x822FFF` | 8,531,192 B | zImage (ARM compressed kernel) |
| `0x823800` – `0x999FFF` | 1,538,048 B | Rockchip Resource Image (RSCE) |

### Kernel (zImage)

- **Format:** ARM zImage (compressed), magic `0x016F2818` at offset +36
- **Decompressed size:** ~8.14 MB
- **Architecture:** ARMv7 (Cortex-A7)
- **gzip stream:** Found at offset `0x6F60` within zImage
- **Kernel version:** 6.1.99 (confirmed from os-release and DTB; version string is inside the compressed payload)

### Second Stage — Rockchip Resource Image (RSCE)

The "second stage" is not a DTB directly — it's a **Rockchip Resource Image** container with magic `RSCE` (`0x52534345`).

| Field | Value |
|---|---|
| Magic | `RSCE` |
| Entry count | 1 |
| Block size | 512 bytes |
| Entry table | Offset 512, 1 entry of 512 bytes |

**Contents:**

| Tag | Filename | Offset | Size | Format |
|---|---|---|---|---|
| `ENTR` | `rk-kernel.dtb` | 2,048 bytes | 122,478 bytes (119.6 KB) | Flattened Device Tree (FDT) |

### Device Tree (rk-kernel.dtb)

| Property | Value |
|---|---|
| Compatible | `rockchip,rv1126-evb-ddr3-v13`, `rockchip,rv1126`, `rockchip,rv1126-flsun` |
| Model | `Rockchip RV1126 EVB DDR3 V2.3 FLSUN-OS mmc` |
| FDT version | 17 |
| Total size | 122,478 bytes |

Notable DTB features:
- 8x PWM controllers (`rv1126-pwm`)
- 5x I2C buses (`rv1126-i2c`)
- 5x UART ports (`rv1126-uart`)
- 2x SPI buses (`rv1126-spi`)
- MIPI DSI display output (`rv1126-mipi-dsi`)
- USB 3.0 DWC3 controller
- NPU via devfreq (`ffbc0000.npu`)
- CAN bus controller (`rockchip,can-1.0`)
- Ethernet GMAC (`rv1126-gmac`)
- 3x eMMC/SD controllers (`rv1126-dw-mshc`)
- ISP camera pipeline (rkcif, rkisp, rkispp)
- Rockchip power management (`rockchip-suspend`, `rv1126-pmu`)

## Boot Sequence

### rc.local Startup

```bash
/etc/init.d/first-boot.sh                           # First boot only (removes itself after)
/home/pi/flsun-os/system/usb-mount-delete-unused.sh &  # Clean stale USB mounts
/usr/bin/input-event-daemon &                        # Power button / touch handler
rfkill block bluetooth                               # Disable Bluetooth
echo performance > .../scaling_governor              # CPU at max frequency
cat .../brightness > /sys/.../brightness             # Restore screen brightness
(sleep 15 && iwconfig wlan0 power off) &             # Disable Wi-Fi power saving
```

### First Boot Script (`/etc/init.d/first-boot.sh`)

The first-boot script runs exactly once and then removes itself from rc.local:

1. **Resize rootfs** — Uses `sgdisk -e` + `parted resizepart` + `resize2fs` to expand the root partition to fill available eMMC space
2. **Set unique hostname** — `FLSUN-S1-XXXX` where XXXX is last 4 chars of wlan0 MAC address
3. **Regenerate SSH keys** — `ssh-keygen -A` + restart SSH
4. **Create symlinks** — `easy-installer` → `installer.sh`, `kiauh` → `kiauh.sh`
5. **Configure web UIs** — Restores Mainsail and Fluidd database entries via Moonraker API (JSON files)
6. **Enable ZRAM swap** — Adds `/dev/zram0` to fstab (512 MB compressed swap)
7. **Self-remove** — Deletes its own invocation line from rc.local
8. **Reboot** — `sleep 5 && reboot`

### NPU Init (`S60NPU_init`)

```bash
insmod /lib/modules/galcore.ko contiguousSize=0x400000
echo $MAX_FREQ > /sys/class/devfreq/ffbc0000.npu/userspace/set_freq
start_rknn.sh &
```

Loads the Vivante GPU/NPU kernel module and starts the RKNN runtime (used for AI camera features, though the open-source edition doesn't actively use it).

### boot.img Creation

The boot.img is created separately (not by debos) using `mkbootimg`:

```bash
mkbootimg \
  --kernel zImage \
  --second resource.img \
  --base 0x10000000 \
  --kernel_offset 0x00008000 \
  --second_offset 0x00F00000 \
  --tags_offset 0x00000100 \
  --pagesize 2048 \
  --output boot.img
```

Where:
- `zImage` is the ARM compressed kernel (compiled from Rockchip's Linux 6.1.99 BSP)
- `resource.img` is the Rockchip resource image (created with `resource_tool` from rkbin) containing `rk-kernel.dtb`

### Kernel Source

The kernel is compiled from the Rockchip BSP (not from mainline Linux). It uses Rockchip's vendor kernel tree for the RV1126, which includes:
- CIF/ISP/ISPP camera pipeline drivers
- NPU driver (galcore.ko built externally)
- RGA2 graphics accelerator
- Rockchip-specific power management
- DRM/display support for MIPI DSI

The device tree adds a custom compatible `rockchip,rv1126-flsun` on top of the standard `rockchip,rv1126-evb-ddr3-v13`.

## Key Findings Summary

1. **Build tool is debos** — confirmed by the ext4 `Last Mounted` path containing `.debos-XXXXXXXX`
2. **boot.img uses Android boot image format** (mkbootimg), not FAT/extlinux as previously theorized
3. **Rockchip Resource Image (RSCE)** packs the DTB, not a raw DTB file
4. **No ramdisk** — the kernel boots directly to the ext4 rootfs, no initramfs
5. **Only 1 kernel module** — `galcore.ko` (NPU driver); everything else is built-in
6. **1,032 Debian packages** installed including full cross-compilation toolchain
7. **Three Python virtualenvs** for Klipper, Moonraker, and KlipperScreen (all Python 3.13)
8. **First-boot script** handles partition resizing, hostname, SSH keys, web UI config, ZRAM
9. **Vendor firmware** directory provides Wi-Fi/BT blobs from Rockchip BSP
10. **Custom DTB** adds `rockchip,rv1126-flsun` compatible string to the standard EVB board

## Open Questions

1. **debos recipe not available** — The actual YAML recipe is not included in the image or the public repositories. Guilouz may keep this private.
2. **Kernel source** — The exact kernel source tree and config are not publicly documented. It's a Rockchip BSP kernel (v6.1.99) with custom DTB modifications.
3. **U-Boot/Trust images** — The eMMC image only contains boot.img and rootfs.img. The bootloader (IDBLoader, U-Boot, Trust) must be either pre-flashed on the SoC or included in the SD card image variant.
4. **SD card image analysis** — The SD image (`FLSUN-OS-S1-SD-3.0.img.gz`, 1.36 GB) likely contains the full GPT layout including bootloader partitions. Not analyzed here.
5. **NPU usage** — The NPU init script loads the driver and starts `start_rknn.sh`, but the Open Source Edition doesn't appear to actively use AI features. The stock FLSUN firmware may use it for camera-based features.
